Chociaz 2M_CODE moze dzialac bezposrednio na Windows, dla najlepszych rezultatow zalecamy [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install). WSL zapewnia srodowisko Linuksa, ktore bardzo dobrze wspolpracuje z funkcjami 2M_CODE.

:::tip[Dlaczego WSL?]
WSL oferuje lepsza wydajnosc systemu plikow, pelna obsluge terminala i zgodnosc z narzedziami developerskimi, na ktorych opiera sie 2M_CODE.
:::

---

## Konfiguracja

<Steps>

1. **Zainstaluj WSL**

   Jesli jeszcze tego nie zrobiles, [zainstaluj WSL](https://learn.microsoft.com/en-us/windows/wsl/install) wedlug oficjalnego przewodnika Microsoft.

2. **Zainstaluj 2M_CODE w WSL**

   Po skonfigurowaniu WSL otworz terminal WSL i zainstaluj 2M_CODE jedna z [metod instalacji](/docs/).

   ```bash
   curl -fsSL https://2M_CODE.ai/install | bash
   ```

3. **Korzystaj z 2M_CODE przez WSL**

   Przejdz do katalogu projektu (pliki Windows sa dostepne przez `/mnt/c/`, `/mnt/d/` itd.) i uruchom 2M_CODE.

   ```bash
   cd /mnt/c/Users/YourName/project
   2M_CODE
   ```

</Steps>

## Aplikacja desktopowa + serwer WSL

Jesli wolisz aplikacje desktopowa 2M_CODE, ale chcesz uruchamiac serwer w WSL:

1. **Uruchom serwer w WSL** z `--hostname 0.0.0.0`, aby zezwolic na polaczenia zewnetrzne:

   ```bash
   2M_CODE serve --hostname 0.0.0.0 --port 4096
   ```

2. **Polacz aplikacje desktopowa** z `http://localhost:4096`

:::note
Jesli `localhost` nie dziala w Twojej konfiguracji, polacz sie przez adres IP WSL (w WSL: `hostname -I`) i uzyj `http://<wsl-ip>:4096`.
:::

:::caution
Przy uzyciu `--hostname 0.0.0.0` ustaw `2M_CODE_SERVER_PASSWORD`, aby zabezpieczyc serwer.

```bash
2M_CODE_SERVER_PASSWORD=your-password 2M_CODE serve --hostname 0.0.0.0
```

:::

## Klient web + WSL

Dla najlepszej pracy w przegladarce na Windows:

1. **Uruchamiaj `2M_CODE web` w terminalu WSL**, a nie w PowerShell:

   ```bash
   2M_CODE web --hostname 0.0.0.0
   ```

2. **Otworz w przegladarce Windows** pod adresem `http://localhost:<port>` (2M_CODE wypisze URL)

Uruchamianie `2M_CODE web` z WSL zapewnia poprawny dostep do systemu plikow i integracje z terminalem, a nadal pozostaje dostepne z przegladarki Windows.

## Dostep do plikow Windows

WSL moze uzyskac dostep do wszystkich plikow Windows przez katalog `/mnt/`:

- Dysk `C:` → `/mnt/c/`
- Dysk `D:` → `/mnt/d/`
- I tak dalej...

Przyklad:

```bash
cd /mnt/c/Users/YourName/Documents/project
2M_CODE
```

:::tip
Aby uzyskac najplynniejsza prace, rozwaz sklonowanie/skopiowanie repozytorium do systemu plikow WSL (np. do `~/code/`) i uruchamianie tam 2M_CODE.
:::

---

## Wskazowki

- Uruchamiaj 2M_CODE w WSL dla projektow zapisanych na dyskach Windows - dostep do plikow jest bezproblemowy
- Uzywaj [rozszerzenia WSL dla VS Code](https://code.visualstudio.com/docs/remote/wsl) razem z 2M_CODE, aby miec zintegrowany workflow
- Konfiguracja 2M_CODE i sesje sa przechowywane w srodowisku WSL pod `~/.local/share/2M_CODE/`
